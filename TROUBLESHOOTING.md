# Руководство по устранению проблем со стилями MoldTool

## Проблема: Стили не отображаются правильно

### Быстрое решение

1. **Очистите кэш браузера:**
   - Chrome/Edge: `Ctrl + Shift + Delete` → Очистить кэш
   - Firefox: `Ctrl + Shift + Delete` → Очистить кэш
   - Или используйте режим инкогнито: `Ctrl + Shift + N`

2. **Перезапустите сервер Django:**
   ```bash
   # Остановите сервер (Ctrl + C)
   # Затем запустите снова:
   python manage.py runserver
   ```

3. **Жёсткая перезагрузка страницы:**
   - `Ctrl + F5` (Windows/Linux)
   - `Cmd + Shift + R` (Mac)

### Проверка статических файлов

1. **Убедитесь что статические файлы собраны:**
   ```bash
   python manage.py collectstatic --noinput
   ```

2. **Проверьте что файлы существуют:**
   - `static/css/main.css`
   - `static/css/variables.css`
   - `static/css/base.css`
   - `static/css/components.css`
   - `static/css/layout.css`
   - `static/js/main.js`

3. **Проверьте настройки в `config/settings.py`:**
   ```python
   STATIC_URL = '/static/'
   STATICFILES_DIRS = [BASE_DIR / 'static']
   STATIC_ROOT = BASE_DIR / 'staticfiles'
   DEBUG = True
   ```

### Проверка загрузки ресурсов

1. **Откройте DevTools в браузере:** `F12`

2. **Перейдите на вкладку Network (Сеть)**

3. **Перезагрузите страницу** и проверьте:
   - Все ли файлы CSS загружаются (статус 200)?
   - Загружается ли Tailwind CSS CDN?
   - Есть ли ошибки 404?

### Проблемы с Tailwind CSS

Проект использует Tailwind CSS через CDN. Если Tailwind не работает:

1. **Проверьте подключение к интернету**

2. **Проверьте в DevTools Console наличие ошибок:**
   - Откройте консоль (`F12` → Console)
   - Ищите красные ошибки

3. **Убедитесь что CDN доступен:**
   - Откройте в браузере: https://cdn.tailwindcss.com
   - Должен загрузиться JavaScript файл

### Проблемы с тёмной темой

Если фон белый вместо тёмного:

1. **Проверьте что в `base.html` правильно указаны классы:**
   ```html
   <body class="font-inter bg-dark text-white antialiased">
   ```

2. **Проверьте что Tailwind config правильно настроен** (в `<script>` теге в `base.html`)

### Проблемы с цветами

Если цвета не те которые ожидаются:

1. **Проверьте `static/css/variables.css`** - там определены все CSS-переменные
2. **Проверьте Tailwind config в `base.html`** - там определены цвета бренда

### Режим разработки

Для отладки стилей рекомендуется:

1. **Включить DEBUG режим** в `config/settings.py`:
   ```python
   DEBUG = True
   ```

2. **Использовать режим инкогнито браузера** для избежания проблем с кэшем

3. **Проверить DevTools Elements** - инспектировать элементы и смотреть какие стили применяются

### Проверка путей к статическим файлам

1. **В шаблоне должен быть `{% load static %}`** в начале файла

2. **Пути должны быть:**
   ```django
   {% static 'css/main.css' %}
   {% static 'js/main.js' %}
   {% static 'images/logo.jpg' %}
   ```

### Если ничего не помогло

1. **Удалите папку `staticfiles` и пересоберите:**
   ```bash
   rmdir /s staticfiles  # Windows
   # или
   rm -rf staticfiles    # Linux/Mac
   
   python manage.py collectstatic --noinput
   ```

2. **Проверьте что сервер запущен правильно:**
   ```bash
   python manage.py runserver 0.0.0.0:8000
   ```
   Откройте: http://127.0.0.1:8000/

3. **Проверьте консоль браузера на JavaScript ошибки**

4. **Попробуйте другой браузер** для исключения проблем с конкретным браузером

## Часто задаваемые вопросы

### Q: Почему некоторые стили работают, а некоторые нет?

A: Скорее всего проблема с кэшем браузера. Используйте `Ctrl + F5` для жёсткой перезагрузки.

### Q: Tailwind классы не работают

A: Проверьте:
1. Загружается ли CDN (DevTools → Network → tailwindcss)
2. Нет ли блокировки со стороны антивируса/файрвола
3. Есть ли интернет

### Q: Страница выглядит "сломанной"

A: Это значит что CSS не загрузился. Проверьте:
1. Статические файлы собраны (`collectstatic`)
2. Пути правильные в шаблонах
3. Сервер запущен
4. Нет ошибок в консоли браузера

### Q: После обновления кода стили не изменились

A: Добавлена версионность статических файлов. Измените версию в `base.html`:
```django
<link rel="stylesheet" href="{% static 'css/main.css' %}?v=1.0.3">
```

2. Ошибками из консоли браузера (F12 → Console)
3. Версией браузера
4. Операционной системой
